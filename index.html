Inicio

HTTP, de sus siglas en inglés: "Hypertext Transfer Protocol", es el nombre de un protocolo el cual nos permite realizar una petición de datos y recursos, como pueden ser documentos HTML. Es la base de cualquier intercambio de datos en la Web, y un protocolo de estructura cliente-servidor, esto quiere decir que una petición de datos es iniciada por el elemento que recibirá los datos (el cliente), normalmente un navegador Web. Así, una página web completa resulta de la unión de distintos sub-documentos recibidos, como, por ejemplo: un documento que especifique el estilo de maquetación de la página web (CSS), el texto, las imágenes, vídeos, scripts, etc...

HTTP utiliza una estructura de solicitud-respuesta. Una solicitud HTTP se envía desde un cliente (por ejemplo, un navegador web) a un servidor, y el servidor responde con los recursos solicitados o un mensaje de error si la solicitud no se puede satisfacer. Aquí están los componentes clave de una solicitud y una respuesta HTTP

¿Qué es TCP/UDP?

El protocolo de control de transmisión (TCP) y el protocolo de datagramas de usuario (UDP) son dos protocolos diferentes que se ejecutan sobre el protocolo de Internet (IP) y se utilizan para la transmisión de datos. Ambos TCP y UDP se utilizan ampliamente en la transmisión de datos a través de Internet. Sin embargo, cada uno tiene sus propias ventajas y desventajas. TCP es un protocolo orientado a la conexión, mientras que UDP es un protocolo sin conexión. TCP es confiable mientras que UDP no es confiable. TCP es lento en comparación con UDP.

Protocolos de red y sus puertos

  1. HTTP (Hypertext Transfer Protocol) - Puerto 80 (por defecto).
  2. HTTPS (Hypertext Transfer Protocol Secure) - Puerto 443 (por defecto).
  3. SMTP (Simple Mail Transfer Protocol) - Puerto 25 (por defecto).
  4. POP3 (Post Office Protocol 3) - Puerto 110 (por defecto).
  5. IMAP (Internet Message Access Protocol) - Puerto 143 (por defecto).
  6. MIME (Multipurpose Internet Mail Extensions) - MIME no utiliza un puerto específico, es una extensión para el formato de correo electrónico.
  7. FTP (File Transfer Protocol) - Puerto 21 (por defecto).
  8. FTPS (File Transfer Protocol Secure) - Puerto 990 (por defecto).
  9. SFTP (SSH File Transfer Protocol) - Puerto 22 (por defecto).
  10. BitTorrent - Utiliza una gama de puertos, generalmente entre 6881 y 6889.
  11. eDonkey - Puerto 4662 (por defecto).
  12. FastTrack (utilizado por Kazaa) - Puerto 1214 (por defecto).
  13. IRC (Internet Relay Chat) - Puerto 6667 (por defecto).
  14. XMPP (Extensible Messaging and Presence Protocol) - Puerto 5222 (por defecto para conexiones no seguras) y 5223 (para conexiones seguras).
  15. MTProto (Protocolo de Telegram) - Utiliza una gama de puertos, generalmente entre 443 y 449.

Software De Aplicación

Que son los Software de Aplicaccion

Un software de aplicación es un programa informático que permite al usuario realizar tareas específicas. Contrasta con el software del sistema, que es el que permite a la computadora funcionar. El software de aplicación usa las capacidades del sistema informático directamente para realizar una tarea determinada. Este tipo de programas se diferencia del software de programación, que es el que se utiliza para integrar y crear los diferentes programas informáticos.

Malware

El malware (abreviatura de "software malicioso") es un tipo de software diseñado para dañar, infectar o robar información de una computadora, dispositivo o red sin el consentimiento del usuario. Puede incluir virus, troyanos, gusanos y otros programas maliciosos que comprometen la seguridad y privacidad de los sistemas informáticos.

Troyanos

Se disfrazan de software legítimo para engañar al usuario y luego permiten el acceso no autorizado a la computadora o dispositivo infectado.

Worms/Gusanos

Son programas maliciosos que se propagan automáticamente a través de redes y sistemas, replicándose y afectando múltiples dispositivos sin la intervención del usuario.

Adware

Muestra anuncios no deseados en el dispositivo infectado, a menudo con fines publicitarios o de lucro, y puede ralentizar el rendimiento del sistema.

Ramsomware

Bloquea o cifra archivos y exige un rescate a cambio de desbloquearlos o descifrarlos, lo que puede causar la pérdida de datos y la extorsión.

Spyware

Recopila información personal del usuario sin su consentimiento, como datos de navegación, contraseñas y actividades en línea, para ser utilizada con fines maliciosos.

Uso Correcto De Las IA's

Mejorar los Promps

Estos son algunos pasos que puedes utilizar para obtener un mejor resultado a la hora de utilizar alguna IA.

  1. Especificar la salida de los datos, ejemplo: Necesito una tabla de top 10 peliculas en este formato "Nombre""Año""Calificacion"
  2. Pedir que el resultado cumpla con las condiciones, Ejemplo: Dame una lista de las capitales de todos los paises europeos. ¿La lista contiene todos los paises europeos?. Al hacer esto nos aseguramos de si la lista esta completa o faltan paises. De ser asi el resultado seria el esperado.
  3. Ofrecer un ejemplo del resultado esperado, Ejemplo: Genera una lista de los juegos mas jugados este año, estructurado de esta forma: "2020 Valorant, Descripcion del video juego".
  4. Pedir al modelo que trabaje en una sola solucion, Ejemplo: 2+2 es 4, no hagas ninguna operacion solo contesta si o no a las siguientes preguntas.
  5. Especificar los pasos para cumplir una tarea, Ejeplo: Necesito un resumen de un la pelicula "El señor de los anillos: Las 2 torres, Primero necesito una introduccion, despues un desarrollo y por ultimo una conclucion."

Retro alimentacion.

Evaluar los resultados y mejorar los promps para obtener mejores resultados. En lo personal creo que esto es lo mas importante a la hora de utilizar un IA, ya que en base a lo que nosotros escribimos y como lo hacemos es la forma en la que la IA ira evolucionando.
